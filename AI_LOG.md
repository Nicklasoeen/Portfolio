# AI Usage Log

This document records a selection of instances where an AI assistant was used during the development of this portfolio project.

---

Tool used: ChatGPT
Date: 28 February 2026
Purpose: Clarification on how to structure a single-page portfolio using JavaScript modules and template literals for rendering HTML content dynamically.
Outcome: Gained a clearer understanding of ES module imports/exports and how to use template literals to build reusable page components.

Tool used: ChatGPT
Date: 28 February 2026
Purpose: Help with "how to create" a smooth mouse-following animation for project card logos using `requestAnimationFrame` and easing.
Outcome: Understood the concept of animation loops with easing values and how `requestAnimationFrame` provides smoother results than CSS transitions for cursor-tracking effects. Implemented the logo follow behavior independently after reviewing the explanation.

Tool used: ChatGPT
Date: 2 March 2026
Purpose: Debugging a z-index issue where project card logos were visible on top of card images before the user hovered over them.
Outcome: Identified that the logo needed its visibility or opacity managed through CSS hover states rather than relying on z-index. Applied the fix independently.

Tool used: ChatGPT
Date: 4 March 2026
Purpose: Guidance on building an accessible accordion component with open/close toggling using JavaScript class manipulation.
Outcome: Learned how toggling an `active` class on accordion items combined with CSS `max-height` transitions creates a expand/collapse effect without external libraries. Built the accordion section for the Fields area.

Tool used: ChatGPT
Date: 4 March 2026
Purpose: CSS layout advice for positioning the project showcase grid and ensuring responsive behavior on smaller screens.
Outcome: Better understanding of CSS Grid with `grid-template-columns: repeat(2, minmax(0, 1fr))` for the two-column layout and how a media query can switch it to a single column on mobile.

Tool used: ChatGPT
Date: 4 March 2026
Purpose: Syntax help with linking to a PDF file from HTML and ensuring it opens in a new tab securely.
Outcome: Clarified the use of `target="_blank"` together with `rel="noopener"` for security when linking external resources, and added this to the CV and Cover Letter download links.
